# Home MBA

Домашняя 12-месячная программа MBA с учебными материалами в Markdown, автоматической проверкой структуры курса и статическим web-интерфейсом для двух независимых контуров:

1. **Готовность материалов курса** — объективное состояние файлов проекта.
2. **Мой учебный прогресс** — личные данные учащегося, хранящиеся только локально в браузере.

## Назначение курса
- 52 недели, 14 модулей, 4 уровня.
- Формат: теория → примеры → кейс → практика → тест → рефлексия.
- Порог завершения модуля: общий результат не ниже 70%, практика не ниже 65%.
- Повторение: 7 / 30 / 90 дней.

## Структура проекта
- `modules/` — учебные модули.
- `assessments/answer_keys/` — ключи ответов, только локально.
- `assessments/grading_rubrics/` — рубрики оценки.
- `resources/` — общие ресурсы.
- `scripts/` — генератор манифеста и проверки.
- `web/` — React + TypeScript + Vite интерфейс.
- `TASKS.md` — backlog проекта.
- `DEVELOPMENT_LOG.md` — журнал изменений.
- `CURRICULUM.md`, `STUDY_PLAN.md`, `COMPETENCY_MAP.md` — базовая методика и структура.
- `PROGRESS_TRACKER.csv` — исходный CSV-трекер.

## Текущий статус
- Всего модулей: 14.
- Фактическая готовность модулей определяется автоматически через `scripts/generate-course-manifest.mjs`.
- На текущем состоянии проекта полностью содержательно разработан модуль 02; остальные модули отображаются по реально найденному наполнению.
- Статусы разработки: `not_started`, `in_progress`, `content_ready`, `ready_for_review`, `verified`.

## Готовые модули
Список и статусы не хардкодятся: UI и проверки берут их из `web/src/data/course-manifest.json`, который генерируется сканером структуры.

## Запуск UI
```bash
cd web
npm install
npm run dev
```

Для production-сборки:
```bash
cd web
npm run build
```

## Запуск проверок
Из корня проекта:
```bash
npm run generate:manifest
npm run validate
```

Проверки включают:
- структуру модулей;
- локальные Markdown-ссылки;
- отсутствие answer keys в web-артефакте;
- корректность манифеста;
- корректность схемы прогресса;
- отсутствие секретов в Git;
- отсутствие локальных файлов прогресса в Git;
- lint, unit tests и build web-приложения.

## Резервное копирование прогресса
Личный прогресс хранится в `localStorage` браузера и **не коммитится** в Git. В UI доступны:
- экспорт прогресса в JSON;
- импорт JSON с проверкой версии схемы;
- сброс прогресса с подтверждением.

## Правила работы с answer keys
- `assessments/answer_keys` не включаются в web-сборку.
- Генератор манифеста сообщает только факт наличия ключа (`hasAnswerKey`).
- При копировании учебных Markdown-файлов в `web/public/content` ссылки на ключи автоматически удаляются/маскируются.
- Если репозиторий станет публичным, answer keys должны оставаться вне публичного контента.

## GitHub и публикация
- Репозиторий по умолчанию должен быть **приватным**.
- Лицензия сейчас: **All rights reserved**.
- GitHub Pages workflow подготовлен, но публикация требует ручного включения Pages и не меняет видимость репозитория.
- Для Pages используется hash-routing, поэтому прямое открытие маршрутов не приводит к необработанной 404.
