# Home MBA

Домашняя 12-месячная программа MBA с учебными материалами в Markdown, автоматической проверкой структуры курса и статическим web-интерфейсом для двух независимых контуров:

1. **Готовность материалов курса** — объективное состояние файлов проекта.
2. **Мой учебный прогресс** — личные данные учащегося, хранящиеся только локально в браузере.

## Назначение курса
- 52 недели, 14 модулей, 4 уровня.
- Формат: обзор → теория → примеры → кейс → практика → тест → рефлексия → ресурсы.
- Порог завершения модуля: общий результат не ниже 70%, практика не ниже 65%.
- Повторение: 7 / 30 / 90 дней.

## Структура проекта
- `modules/` — учебные модули.
- `assessments/answer_keys/` — ключи ответов, только локально.
- `assessments/grading_rubrics/` — рубрики оценки.
- `resources/` — общие ресурсы.
- `scripts/` — генератор манифеста и проверки.
- `web/` — React + TypeScript + Vite интерфейс.
- `TASKS.md` — backlog проекта.
- `DEVELOPMENT_LOG.md` — журнал изменений.
- `CURRICULUM.md`, `STUDY_PLAN.md`, `COMPETENCY_MAP.md` — базовая методика и структура.
- `PROGRESS_TRACKER.csv` — исходный CSV-трекер.

## Текущий статус
- Всего модулей: 14.
- Фактическая готовность модулей определяется автоматически через `scripts/generate-course-manifest.mjs`.
- На текущем состоянии проекта содержательно разработан модуль 02; модуль 01 и остальные отображаются по реально найденному наполнению.
- Статусы разработки: `not_started`, `in_progress`, `content_ready`, `ready_for_review`, `verified`.
- Схема пользовательского прогресса: `schemaVersion = 2` с миграцией со старой версии.

## Учебный режим / course player
Приложение поддерживает полноценный режим прохождения модулей по маршруту `#/learn/:moduleSlug/:unitId`.

Что умеет player:
- открывать реальные Markdown-материалы из `modules/*` через безопасную публикацию в `web/public/content`;
- строить учебные units из файлов и стабильных заголовков `##` / `###`;
- сохранять последний урок, время посещения и позицию чтения;
- поддерживать заметки, закладки, черновики кейса и практики;
- вести безопасные попытки теста без публикации answer keys;
- завершать модуль только при выполнении пороговых правил.

## Запуск проверок
Из корня проекта:
```bash
npm run generate:manifest
npm run validate
```

Проверки включают:
- структуру модулей;
- локальные Markdown-ссылки;
- отсутствие answer keys в web-артефакте;
- корректность манифеста;
- корректность схемы прогресса;
- отсутствие секретов в Git;
- отсутствие локальных файлов прогресса в Git;
- lint, unit tests и build web-приложения.

## Хранение и резервное копирование прогресса
Личный прогресс хранится в `localStorage` браузера и **не коммитится** в Git. В UI доступны:
- экспорт прогресса в JSON;
- импорт JSON с проверкой версии схемы и подтверждением замены;
- сброс прогресса с подтверждением.

### Что входит в экспортируемую схему
- статусы модулей и оценки;
- последний урок и позиции чтения;
- статусы unit-уровня;
- заметки и закладки;
- черновики кейса и практики;
- история попыток теста;
- ответы рефлексии;
- даты повторения.

## Безопасное прохождение теста
- `assessments/answer_keys` не включаются в web-сборку.
- Генератор манифеста сообщает только факт наличия ключа (`hasAnswerKey`).
- В публичном UI сохраняются ответы пользователя, флаги и история попыток, но **не** правильные ответы.
- В репозиторий не коммитятся пользовательские попытки, локальные результаты проверки и прогресс браузера.
- Следующий шаг для локального офлайн-grading — отдельный CLI-скрипт, который можно добавить позже без публикации ключей.

## Правила работы с answer keys
- `assessments/answer_keys` не включаются в web-сборку.
- При копировании учебных Markdown-файлов в `web/public/content` ссылки на ключи автоматически удаляются/маскируются.
- Если репозиторий публичный, answer keys всё равно остаются вне публичного контента.
