# QUALITY REPORT

## Итог на текущий момент
- Проект подготовлен к отдельному Git-репозиторию.
- Создан безопасный `.gitignore`.
- Реализован генератор манифеста курса.
- Реализован статический UI без backend.
- Личный прогресс хранится локально и не должен попадать в Git.
- Answer keys не включаются в manifest JSON и не должны попадать в публичную сборку.

## Что проверено
- `npm run generate:manifest`
- `cd web && npm run lint`
- `cd web && npm run test`
- `cd web && npm run build`

## Найденные риски
- В исходных markdown-файлах могут встречаться упоминания путей к answer keys; генератор санитизирует их при публикации в `web/public/content`.
- `gh` не установлен, поэтому создание remote и push пока не автоматизированы из этой среды.
- Размер production bundle превышает warning-порог Vite.

## Рекомендации
- После установки `gh` повторить `npm run validate`, затем создать приватный репозиторий и выполнить push.
- Следующим шагом уменьшить размер bundle через lazy-loading страниц и markdown-блоков.
